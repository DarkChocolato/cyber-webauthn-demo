Prompt 1: เริ่มต้นค้นคว้าหาไอเดียและการออกแบบ Mock Backend
คำสั่ง (Prompt):
"ช่วยแนะนำวิธีการพัฒนาระบบสมัครสมาชิกและล็อกอิน FIDO2 / WebAuthn โดยต้องการนำเสนอเป็นลักษณะ Demo แบบไม่มี Backend จริง (ใช้ LocalStorage ในเบราว์เซอร์จำลองฐานข้อมูลการลงทะเบียนและการตรวจสอบ) เพื่อเอาไปนำเสนองานวิชาความปลอดภัยทางคอมพิวเตอร์ และเน้นความสวยงามสไตล์ Cyber/Glassmorphism"
ผลลัพธ์ที่ได้: AI แนะนำโครงสร้าง Single Page App และวิธีการแปลงค่า ArrayBuffer ไปเป็น Base64URL ด้วยฟังก์ชัน Javascript เพื่อบันทึกลงใน localStorage และให้คำแนะนำเรื่องข้อจำกัดที่ WebAuthn ต้องทำงานบน HTTPS หรือ localhost เท่านั้น
Prompt 2: ขอฟังก์ชันการเข้ารหัสข้อมูลสำหรับส่งเข้าระบบ FIDO2
คำสั่ง (Prompt):
"เขียนฟังก์ชัน JavaScript สำหรับเข้ารหัสและถอดรหัส ArrayBuffer เป็น Base64 และ Base64URL เพื่อใช้เป็นตัวช่วย (helper) ในการส่งข้อมูลขึ้นลงระหว่างเบราว์เซอร์กับฟังก์ชัน WebAuthn API (navigator.credentials.create และ navigator.credentials.get)"
ผลลัพธ์ที่ได้: ได้ฟังก์ชันช่วยแปลงชนิดข้อมูล (ArrayBuffer <-> Base64URL) ในไฟล์ helper ซึ่งเป็นส่วนสำคัญที่สุดเนื่องจาก API มาตรฐานของ WebAuthn รับข้อมูลเป็นชนิด ArrayBuffer เท่านั้น ซึ่งไม่สามารถเซฟลง JSON ตรงๆ ได้
Prompt 3: ขอโค้ดการทำ FIDO2 Registration และ Authentication
คำสั่ง (Prompt):
"เขียนโค้ด JavaScript ในฝั่งของ Client โดยเรียกใช้ navigator.credentials.create() สำหรับสมัครใช้งานด้วยลายนิ้วมือ และ navigator.credentials.get() สำหรับการยืนยันตัวตนล็อกอิน โดยใช้ข้อมูลจาก LocalStorage ในการเก็บและดึงข้อมูล Credential มาตรวจสอบ"
ผลลัพธ์ที่ได้: ได้รับสคริปต์หลักในการดึงข้อมูลจากฟอร์ม, กำหนดค่าตัวเลือก publicKey (เช่น rp, user, challenge, pubKeyCredParams) และเรียกใช้เบราว์เซอร์พ็อปอัพเพื่อสแกนนิ้วมือ/ใบหน้า

💡 คำอธิบายทางเทคนิคและความปลอดภัย (Security Context)

การทำงานในโปรเจกต์นี้

  • การเข้ารหัสข้อมูล: ข้อมูลไบนารี (ArrayBuffer) จากชิปความปลอดภัยถูกแปลงเป็น Base64 เพื่อจัดเก็บลงใน LocalStorage ของเบราว์เซอร์
  • การลงทะเบียน: อุปกรณ์เครื่องจะสร้างคู่กุญแจ (Public/Private Key pair) โดยกุญแจสาธารณะจะเก็บไว้บนเครื่องจำลอง (LocalStorage) และกุญแจส่วนตัวอยู่ในชิปฮาร์ดแวร์ของผู้ใช้
  • การยืนยันตัวตน: เมื่อผู้ใช้เข้าสู่ระบบ อุปกรณ์ของผู้ใช้จะลงลายมือชื่อดิจิทัล (Digital Signature) บนข้อมูล Challenge แล้วส่งกลับมาตรวจสอบความสอดคล้อง

การนำไปประยุกต์ใช้ในระบบจริง

  • Backend จริง: ฝั่งเซิร์ฟเวอร์จะต้องใช้ไลบรารีช่วยตรวจสอบ เช่น fido2-lib (Node.js) หรือ WebAuthn SDK ของภาษาอื่นๆ
  • การทวนสอบ (Verification): Backend จะสร้าง Challenge ที่ปลอดภัยแบบสุ่มจริง และทวนสอบลายเซ็นที่ส่งกลับมาจากเบราว์เซอร์ด้วยกุญแจสาธารณะที่บันทึกไว้ในฐานข้อมูล
  • ความปลอดภัย: ป้องกันการโจมตีแบบ Replay Attack และการทำ Phishing ได้ 100% เพราะกุญแจจะผูกกับชื่อโดเมน (Relying Party ID) เสมอ
