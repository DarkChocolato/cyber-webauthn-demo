อธิบายการทำงานอย่างละเอียด (System Flow)

ไดอะแกรมและคำอธิบายขั้นตอนการลงทะเบียนและการเข้าสู่ระบบ FIDO2 / WebAuthn

ระบบยืนยันตัวตนไร้รหัสผ่าน FIDO2/WebAuthn จะทำงานในลักษณะของ Asymmetric Cryptography (การเข้ารหัสแบบไม่สมมาตรกุญแจคู่) โดยอุปกรณ์ของผู้ใช้จะเก็บกุญแจส่วนตัว (Private Key) ไว้ในชิปความปลอดภัยอย่างมิดชิด (Secure Enclave / TPM) และฐานข้อมูลจะเก็บเฉพาะกุญแจสาธารณะ (Public Key)

1. ขั้นตอนลงทะเบียน (Registration)

1
ป้อนข้อมูลผู้ใช้
ผู้ใช้กรอกข้อมูล Username, ชื่อจริง, นามสกุล และเบอร์โทรศัพท์บนแบบฟอร์มให้ครบถ้วน
2
สร้างรหัสท้าทาย (Challenge)
ระบบตรวจสอบว่า Username ไม่ซ้ำ และสุ่มสร้างข้อมูลรหัสท้าทาย (Challenge) ขนาด 32 ไบนารีเพื่อใช้ตรวจสอบและป้องกันการโจมตีซ้ำ (Replay Attack)
3
เรียกใช้งาน WebAuthn API
เบราว์เซอร์สั่งทำงานคำสั่ง navigator.credentials.create() พร้อมส่งค่าตัวเลือกการสร้างกุญแจ FIDO2 ไปยังอุปกรณ์ของผู้ใช้
4
ยืนยันตัวตนบนอุปกรณ์
ระบบปฏิบัติการแสดงหน้าต่างแจ้งเตือนเพื่อขอสแกนนิ้วมือ (Fingerprint), ใบหน้า (Face ID) หรือป้อน PIN ของอุปกรณ์
5
ส่งคืนกุญแจสาธารณะ
เมื่อผู้ใช้สแกนผ่าน ชิปฮาร์ดแวร์ความปลอดภัยจะสร้างคู่กุญแจ และส่งข้อมูลกุญแจสาธารณะ (Public Key) และ Credential ID กลับมายังเบราว์เซอร์
6
เข้ารหัสและบันทึกข้อมูล
เบราว์เซอร์แปลงข้อมูลแบบ ArrayBuffer ให้กลายเป็น Base64URL เพื่อให้จัดเก็บโปรไฟล์และ Credential ID ลงใน LocalStorage ได้สำเร็จ

2. ขั้นตอนเข้าสู่ระบบ (Authentication)

1
ป้อนชื่อผู้ใช้งาน
ผู้ใช้กรอกเฉพาะ Username ที่เคยใช้ลงทะเบียนเพื่อเข้าสู่ระบบไร้รหัสผ่าน
2
ดึงข้อมูล Credential ID
ระบบจำลองค้นหาโปรไฟล์ของ Username และดึงข้อมูล Credential ID ที่ลงทะเบียนไว้ขึ้นมาจาก LocalStorage
3
สร้าง Challenge รอบใหม่
ระบบสุ่มสร้างข้อมูลรหัสท้าทาย (Challenge) ชุดใหม่ขึ้นมา เพื่อความปลอดภัยในรอบนั้น ๆ
4
เรียกสแกนนิ้วทวนสอบ
เบราว์เซอร์สั่งทำงานคำสั่ง navigator.credentials.get() โดยระบุ Credential ID ของผู้ใช้เพื่อให้ชิปความปลอดภัยเรียกทำงาน
5
ลงลายมือชื่อดิจิทัล
เมื่อผู้ใช้สแกนนิ้วผ่าน ชิปความปลอดภัยจะหยิบกุญแจส่วนตัว (Private Key) ในเครื่องมาเซ็นลายมือชื่อดิจิทัลกำกับข้อมูล Challenge แล้วส่งคืนเบราว์เซอร์
6
ตรวจสอบลายมือชื่อสำเร็จ
หน้าเว็บเบราว์เซอร์ตรวจสอบความสอดคล้องสำเร็จ แล้วดึงรายละเอียดชื่อ นามสกุล และเบอร์โทรศัพท์ไปแสดงบนหน้าล็อกอินสำเร็จทันที

Prompt 2: ขอฟังก์ชันการเข้ารหัสข้อมูลสำหรับส่งเข้าระบบ FIDO2
คำสั่ง (Prompt):
"เขียนฟังก์ชัน JavaScript สำหรับเข้ารหัสและถอดรหัส ArrayBuffer เป็น Base64 และ Base64URL เพื่อใช้เป็นตัวช่วย (helper) ในการส่งข้อมูลขึ้นลงระหว่างเบราว์เซอร์กับฟังก์ชัน WebAuthn API (navigator.credentials.create และ navigator.credentials.get)"
ผลลัพธ์ที่ได้: ได้ฟังก์ชันช่วยแปลงชนิดข้อมูล (ArrayBuffer <-> Base64URL) ในไฟล์ helper ซึ่งเป็นส่วนสำคัญที่สุดเนื่องจาก API มาตรฐานของ WebAuthn รับข้อมูลเป็นชนิด ArrayBuffer เท่านั้น ซึ่งไม่สามารถเซฟลง JSON ตรงๆ ได้

การทำงานในโปรเจกต์นี้

  • การเข้ารหัสข้อมูล: ข้อมูลไบนารี (ArrayBuffer) จากชิปความปลอดภัยถูกแปลงเป็น Base64 เพื่อจัดเก็บลงใน LocalStorage ของเบราว์เซอร์
  • การลงทะเบียน: อุปกรณ์เครื่องจะสร้างคู่กุญแจ (Public/Private Key pair) โดยกุญแจสาธารณะจะเก็บไว้บนเครื่องจำลอง (LocalStorage) และกุญแจส่วนตัวอยู่ในชิปฮาร์ดแวร์ของผู้ใช้
  • การยืนยันตัวตน: เมื่อผู้ใช้เข้าสู่ระบบ อุปกรณ์ของผู้ใช้จะลงลายมือชื่อดิจิทัล (Digital Signature) บนข้อมูล Challenge แล้วส่งกลับมาตรวจสอบความสอดคล้อง

การนำไปประยุกต์ใช้ในระบบจริง

  • Backend จริง: ฝั่งเซิร์ฟเวอร์จะต้องใช้ไลบรารีช่วยตรวจสอบ เช่น fido2-lib (Node.js) หรือ WebAuthn SDK ของภาษาอื่นๆ
  • การทวนสอบ (Verification): Backend จะสร้าง Challenge ที่ปลอดภัยแบบสุ่มจริง และทวนสอบลายเซ็นที่ส่งกลับมาจากเบราว์เซอร์ด้วยกุญแจสาธารณะที่บันทึกไว้ในฐานข้อมูล
  • ความปลอดภัย: ป้องกันการโจมตีแบบ Replay Attack และการทำ Phishing ได้ 100% เพราะกุญแจจะผูกกับชื่อโดเมน (Relying Party ID) เสมอ
